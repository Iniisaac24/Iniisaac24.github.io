The Problem

Managing risk you can actually see

Most retail investors track returns obsessively but don't measure risk in any structured way. A portfolio that has outperformed the S&P 500 by 3% might have done so by taking on significantly more downside exposure — which only becomes visible during a correction.

This dashboard was built to answer the questions a risk-aware investor asks every month: Am I being compensated for the risk I'm taking? Where is my concentration risk hiding? What's my realistic worst-day scenario?

Key finding: Three stock pairs with correlation above 0.85 identified — all within the tech sector. The portfolio's apparent diversification across 15 names was masking significant concentration risk in a single factor.

Process

From price data to risk dashboard

01

Live data ingestion

yfinance pulls 3 years of adjusted closing prices for all 15 tickers plus the S&P 500 benchmark. The risk-free rate is pulled directly from FRED's DTB3 series (3-month Treasury Bill) — refreshed daily. No static CSV files anywhere in the pipeline.

02

Returns and annualisation

Daily percentage returns calculated with pct_change(). Portfolio daily return is the weighted sum across all holdings (equal-weight by default, adjustable in the dashboard). Annualised using 252 trading days.

03

Risk metrics

Sharpe Ratio uses the live T-Bill rate as the risk-free rate. VaR calculated by historical simulation — the 5th percentile of actual daily returns. Max Drawdown computed as the largest percentage fall from any rolling peak in the period.

04

Correlation analysis

Full pairwise correlation matrix across all holdings. Any pair with correlation above 0.85 is flagged as concentration risk. This is visualised as a heatmap with the diagonal masked for clarity.

05

Streamlit deployment

The entire pipeline runs inside a Streamlit app deployed on Streamlit Community Cloud — free hosting, auto-deploys from the GitHub repo on every push. Data refreshes on page load.

Key Code

Sharpe Ratio + VaR calculation

import yfinance as yf
import numpy as np
import pandas as pd

# Pull risk-free rate from FRED
fred_url = "https://fred.stlouisfed.org/graph/fredgraph.csv?id=DTB3"
rf_annual = pd.read_csv(fred_url, index_col='DATE').replace('.', np.nan).astype(float).ffill().iloc[-1, 0] / 100
rf_daily  = rf_annual / 252

# Portfolio daily returns (equal-weight)
prices       = yf.download(tickers, period='3y', auto_adjust=True, progress=False)['Close']
daily_rets   = prices.pct_change().dropna()
port_daily   = daily_rets.mean(axis=1)   # equal weight

# Sharpe Ratio
excess       = port_daily - rf_daily
sharpe       = (excess.mean() / excess.std()) * np.sqrt(252)

# Value at Risk — Historical Simulation
var_95       = np.percentile(port_daily, 5)

# Max Drawdown
cum_ret      = (1 + port_daily).cumprod()
max_drawdown = ((cum_ret - cum_ret.cummax()) / cum_ret.cummax()).min()

print(f"Sharpe: {sharpe:.3f}  |  VaR(95%): {var_95:.2%}  |  Max DD: {max_drawdown:.2%}")

Findings

What the dashboard revealed

The portfolio outperformed the S&P 500 over the 3-year period, but with a Beta of 1.12 — meaning it amplified market moves. The Sharpe Ratio of 1.24 is above the commonly cited 1.0 threshold, suggesting the excess return justified the additional risk.

Three stock pairs flagged for concentration risk: AAPL↔MSFT (0.91), AAPL↔NVDA (0.87), MSFT↔GOOGL (0.86). Despite having 15 names, the tech sector behaves like a single factor in downturns. Recommended action: replace one tech holding with a low-correlation asset.

VaR of −1.8% at 95% confidence means in the worst 5% of trading days, the portfolio loses at least 1.8% of its value. The CVaR (conditional VaR) is −2.6% — this is the average loss on those worst days, giving a more conservative stress scenario.

The maximum drawdown of −18.3% occurred during the 2022 rate-hike cycle. The portfolio recovered faster than the benchmark due to energy sector holdings (XOM, CVX) which appreciated during the inflationary period — demonstrating meaningful sector diversification despite the tech concentration.
